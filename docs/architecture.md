# UNGRUND — App architecture

> Живой модульный SuperCollider-скелет системы UNGRUND. Каждый синт-слой
> и модуль итерируется ОТДЕЛЬНО. Скелет загружается и звучит из одного
> файла, оставляя чистый сервер после Cmd-. .

---

## 1. Что такое UNGRUND (кратко)

UNGRUND — система абсолютной электронной музыки после Титченса (Tietchens).
Звук трактуется как *субстрат*, а не носитель: весь материал порождается
синтезом из первопринципов (никаких записей/сэмплов, Аксиома II),
композиция строится *блоками*, а не аркой (Аксиома IV), и сама структура
выставляется уху как измерение, а не как выражение. Связующая аксиома:
«ни один звук не оправдан чувством; каждый звук оправдан своей позицией в
калибруемом процессе, слышимом ухом».

Словарь системы — восемь звуковых классов, названных греческими буквами
`alpha`…`theta` (по порядку §C в `docs/reference_v2.md`):

| Класс    | §C   | Премиса (исходное имя UNGRUND)        |
|----------|------|---------------------------------------|
| `alpha`   | §C.1 | sustained sine / Grund (`monade`)     |
| `beta`    | §C.2 | broadband impulse (`riß`)             |
| `gamma`   | §C.3 | granular cluster of sines (`flockung`)|
| `delta`   | §C.4 | narrow-band noise (`binde`)           |
| `epsilon` | §C.5 | drifting comb-filtered noise (`schliere`) |
| `zeta`    | §C.6 | dry granulated noise burst (`kratzer`)|
| `eta`     | §C.7 | composed silence (`zäsur`)            |
| `theta`   | §C.8 | ultrasonic shimmer / ceiling (`firnis`)|

В коде и в этом документе слои называются строго греческими буквами; исходные
немецкие имена (`monade`, `riß`, …) сохраняются только как ссылка на §C
`docs/reference_v2.md`, где описана премиса синтеза каждого класса.
Тембральная редакция 2026 (§K) допускает реверб и делей как операции синтеза
(не как «комнату»): тёплый, обволакивающий, внутренне подвижный материал.

Имена слоёв — ASCII (греческие буквы записаны латиницей: `alpha`, `beta`, …),
поэтому имя файла, символ `Ndef(\<class>)` и спецификация `~spec_<class>`
совпадают (`alpha.scd` → `Ndef(\alpha)` → `~spec_alpha`).

## 2. Семислойная модель приложения

Движок — **JITLib** (`Ndef`/`Tdef`/`MIDIdef`): каждый слой/модуль можно
горячо заменить и итерировать независимо, просто перевыполнив его файл.

| № | Файл | Слой | Ответственность |
|---|------|------|-----------------|
| 1 | `App/1_app.scd` | **app** | Оркестратор + ЕДИНАЯ ПАНЕЛЬ (`~cfg_app`). Boot сервера, база-директория, аллокация шин, ВЫБОР загружаемых слоёв (`layers`), ФЛАГИ включения модулей (`enable*`), авто-пресеты, загрузка в порядке зависимостей, `s.sync`, открытие окон (микшер/виз), старт, мастер-очистка. |
| 2 | `App/2_mixer.scd` | **mixer** | Маршрутизация ТОЛЬКО активных слоёв (`~activeLayers`) в `~busDry`, агрегатор посылов в `~busFxSend`, мастер (LeakDC + Limiter). Сеттеры `~mixSet/~masterSet/~mixSolo/~mixUnsolo` + **окно микшера** `~openMixer` (фейдеры vol/send, solo, кнопка tuner, мастер). |
| 3 | `App/3_lfo.scd` | **lfo** | *Опционально* (`enableLfo`). Центральный реестр автоматизации. Таблица строк (target / param / shape / rate / range); движок создаёт kr-Ndef на строку и маппит на цель. |
| 4 | `App/4_sequencer.scd` | **sequencer** | *Опционально* (`enableSequencer`). Блочный таймлайн + Tdef на TempoClock. Старт/стоп; экспортирует `~seqCurrentBlock`/`~seqStartTime` для виз-окна. |
| 5 | `App/5_control.scd` | **control** | *Опционально* (`enableControl`). MIDI CC→цель; обобщённый MIDIdef. Nil-безопасен к выключенному LFO (`~lfoKeys`). |
| 6 | `App/6_effects.scd` | **effects** | *Опционально* (`enableEffects`). Эффект-прокси: tape reverb (§K.1) + ping-pong delay (§K.2). Читают `~busFxSend`, пишут `~busFxReturn`. |
| 7 | `App/7_viz.scd` | **viz** | *Опционально* (`enableVizGui`). Окно визуализации: таймлайн секвенсора (блоки + playhead) и живые значения LFO. Полностью nil-безопасно при выключенных seq/LFO. |
| — | `App/synths/*.scd` | **synths** | Восемь звуковых классов, по одному файлу. Каждый — `Ndef(\<class>)` с CONFIG-блоком сверху. |

Эффекты концептуально — седьмой слой; нумерация файлов (1–7 + synths/)
отражает *концептуальный* порядок пользователя, тогда как `1_app.scd`
грузит их в *порядке зависимостей* (см. §5).

**ФАЗА САУНД-ДИЗАЙНА (умолчания `~cfg_app`).** Сейчас задача — отстроить все
параметры всех звуков в микшере и закончить саунд-дизайн ДО работы над
секвенсором/автоматизацией/MIDI. Поэтому по умолчанию: эффекты + микшер +
окно микшера ВКЛ; LFO, секвенсор, MIDI, автозапуск — ВЫКЛ. На голом запуске
`1_app.scd` все слои звучат сразу на уровнях микшера, открыто окно сведе́ния.
Включить любой модуль = один флаг `enable*` (или `autostart`) в `~cfg_app`.

## 4. Сигнал-флоу

```
   synths (Ndef)            mixer routers            mixer master         hardware
 ┌───────────────┐        ┌──────────────┐                                  out 0/1
 │ Ndef(\alpha) │──ar──▶ │ Ndef(\mix_*) │──vol──▶ ~busDry ───┐
 │ Ndef(\beta)   │        └──────────────┘                    │
 │ Ndef(\gamma)        ┌──────────────┐                    ▼
 │ Ndef(\delta)  │──ar──▶ │ Ndef(\fxsend)│         ┌────────────────────┐
 │ Ndef(\epsilon)        │  Σ send_<L>  │─Replace▶ │   Ndef(\master)    │──▶ out
 │ Ndef(\zeta)│        └──────────────┘ ~busFxSend  dry + fxReturn,    │
 │ Ndef(\eta) │                │            │    LeakDC, Limiter(0.95) │
 │ Ndef(\theta) │                ▼            │    └────────────────────┘
 └───────────────┘        ┌──────────────┐     │             ▲
                          │Ndef(\tape_reverb)──┘             │
                          │Ndef(\ping_delay) │──▶ ~busFxReturn┘
                          └──────────────┘
```

- Каждый синт-слой выводит свой *сухой* сигнал в свой внутренний Ndef-bus.
  Микшер читает его дважды: роутер `Ndef(\mix_<L>)` пишет масштабированный
  на `vol` сигнал в `~busDry`; агрегатор `Ndef(\fxsend)` суммирует все слои,
  масштабированные на `send_<L>`, и пишет в `~busFxSend` через `ReplaceOut`
  (РОВНО один писатель — шина не накапливается между блоками).
- Эффекты читают `~busFxSend` и пишут в `~busFxReturn`.
- Мастер суммирует `~busDry + ~busFxReturn`, ставит `LeakDC` + `Limiter(0.95)`
  (§K.4), выводит в аппаратные каналы 0/1.

Все межслойные шины аллоцированы через `Bus.audio(s, 2)` в `1_app.scd`:
`~busDry`, `~busFxSend`, `~busFxReturn`. Никаких магических индексов.

## 5. Порядок загрузки (зависимости)

`1_app.scd` грузит так (с `s.sync` между батчами):

```
synths/*.scd  →  6_effects.scd  →  2_mixer.scd  →  3_lfo.scd  →  5_control.scd  →  4_sequencer.scd  →  7_viz.scd
```

Почему: синты и эффекты должны быть определены ДО того, как микшер их
маршрутизирует; lfo и control — ПОСЛЕ микшера (они правят его параметры);
секвенсор — ПОСЛЕДНИМ (он дирижирует громкостями слоёв через микшер);
виз-окно — самым последним (читает данные seq/LFO).

**Гейтинг по флагам.** Каждый модуль КРОМЕ synths грузится только если его
флаг `enable*` в `~cfg_app` истинен. Порядок сохраняется, пропущенные шаги
просто не выполняются. `~activeLayers` (список реально загруженных слоёв)
ставит `1_app.scd` сразу после загрузки synths — микшер и виз читают именно
его, поэтому подмножество `~cfg_app.layers = [...]` не тянет пустые Ndef'ы.
Выключенные модули НИЧЕГО не ломают: единственные внешние читатели —
`~isMapped` (0_lib, nil-guard на `~cfg_lfo`) и `~lfoKeys` (5_control, nil-guard).

## 6. Конвенция «CONFIG сверху»

Каждый файл начинается с явно отделённой секции
`// ===== CONFIG — edit here =====` (обычно Event/словарь правимых
значений), за которой идёт секция `// ===== LOGIC =====`. Поведение
меняется правкой ВЕРХА файла без касания логики.

> **ВАЖНО (структура файла).** Весь файл — РОВНО ОДИН верхнеуровневый блок
> `( … )`: секции CONFIG и LOGIC живут ВНУТРИ одного блока. Это обязательно,
> потому что `1_app.scd` грузит модули через `.load`, а `.load`/интерпретатор
> выполняет только ПЕРВЫЙ блок файла — два отдельных `( )`-блока приведут к
> тому, что LOGIC молча не выполнится (а целиком файл вообще не
> скомпилируется: «unexpected '(', expecting end of file»). При итерации НЕ
> оборачивайте CONFIG и LOGIC в отдельные скобки. Исключение — `1_app.scd`:
> там блок CLEANUP внизу намеренно закомментирован (ручной сброс).

Примеры:

- **mixer** (`2_mixer.scd`): таблица `~cfg_mixer.levels`, ключ = имя слоя,
  значения `( vol: …, send: … )`. Например `alpha: ( vol: 0.85, send: 0.55 )`.
- **lfo** (`3_lfo.scd`): список `~cfg_lfo.lines`, строка =
  `( target: \alpha, param: \amp, shape: \noise, rate: 0.03, min: 0.16, max: 0.24 )`.
  Добавить автоматизацию = добавить строку.
- **control** (`5_control.scd`): `~cfg_ctrl.map`, строка =
  `( cc: 1, type: \mixVol, target: \alpha, min: 0.0, max: 1.0 )`.
  Типы: `\layer` / `\mixVol` / `\mixSend` / `\lfoRate` / `\master`.
- **sequencer** (`4_sequencer.scd`): `~cfg_seq.timeline`, блок =
  `( t: 4, active: [\theta], gone: [], set: [[\delta, \freq, 188]] )`.
- **synths** (`synths/*.scd`): два уровня CONFIG.
  - *заглушки* — `~cfg_<class>` — ключевые параметры класса с разумными
    умолчаниями (из §C / §K.4), пара `Ndef.set`.
  - *слои с панелью управления* (`alpha`, `zeta`) — `~spec_<class>`:
    ЕДИНЫЙ источник правды (список параметров: `key`/`group`/`spec`-диапазон+
    дефолт, опц. `unit`/`disp`). Из него 0_lib строит слайдеры
    (`~openTuner.(\<class>)`), два рандомайзера по группам (`\freq` —
    «частоты», `\tone` — «параметры») и round-trip-пресеты
    (`~savePreset`/`~loadPreset`, `presets/<class>/`). Слой регистрирует свой
    spec в `~ungSpecs` и применяет дефолты. Расширить управление = добавить
    строку в `~spec_<class>` И аргумент в `Ndef`. Окно тюнера безопасно к
    повторному открытию (best-effort закрытие старого окна, onClose чистит
    реестр только если окно ещё «своё»).

## 7. Как загрузить

1. Открыть `App/1_app.scd` в IDE SuperCollider.
2. Выделить весь файл (Cmd-A) и выполнить (Shift-Return). С умолчаниями это:
   - бутстрапит сервер,
   - аллоцирует шины,
   - грузит synths/* → effects → mixer (LFO/control/sequencer ВЫКЛ),
   - авто-грузит пресет `_default` каждого слоя (если есть),
   - открывает ОКНО МИКШЕРА — все слои звучат на уровнях сведения.
3. Остановить звук — **Cmd-.** (безопасно, оставляет чистый сервер).
4. Полная очистка (шины/Ndef/Tdef/MIDIdef/окна) — выполнить нижний абзац
   `// CLEANUP` в `1_app.scd`, либо `~appCleanup.value(true)`.

**Окна вручную:** `~openMixer.value` (микшер), `~openTuner.(\alpha)`
(отстройка слоя), `~openViz.value` (виз seq+LFO).

**Режимы через `~cfg_app`:**
- только тестируемые слои: `~cfg_app.layers = [\alpha, \theta]` (nil = все);
- включить автоматизацию: `~cfg_app.enableLfo = true`;
- включить таймлайн: `~cfg_app.enableSequencer = true` (+ `autostart = true`
  для авто-старта, либо вручную `~seqStart.value` / `~seqStop.value`);
- включить MIDI: `~cfg_app.enableControl = true`;
- виз-окно: `~cfg_app.enableVizGui = true` (нужно с enableLfo/enableSequencer).

## 8. Как итерировать каждый модуль отдельно

Любой файл идемпотентен при перезагрузке (Ndef/Tdef/MIDIdef-ключи
заменяются, не стакаются). Поэтому:

- **Один синт-слой** (например `beta`): открыть `App/synths/beta.scd`,
  поправить CONFIG и/или тело `Ndef(\beta)`, выполнить весь файл (Cmd-A,
  Shift-Return) при работающем приложении. Новый звук подменится с
  `fadeTime`-кроссфейдом; маршрутизация микшера сохранится.
- **Сведение**: править таблицу в `2_mixer.scd`, перевыполнить файл — или
  на лету `~mixSet.(\alpha, \vol, 0.5)` / `~mixSet.(\theta, \send, 0.8)`
  / `~masterSet.(0.8)`.
- **Отстройка одного слоя в полном тракте** (через реверб + мастер): сначала
  `~seqStop.value` (чтобы секвенсор не перебивал громкости), затем
  `~mixSolo.(\alpha)` — звучит только этот слой, остальные глушатся
  (`vol=0`, `send=0`). Крутить параметры через `Ndef(\alpha).set(...)` и
  `~mixSet.(...)`. Вернуть всех — `~mixUnsolo.value` (восстанавливает уровни
  из `~cfg_mixer`, исходное сведение не теряется). Чтобы слой не «голым»
  стартовал, а через тракт, проект запускают целиком с `~cfg_app.autostart =
  false` (см. §7).
- **Автоматизация**: добавить строку в `~cfg_lfo.lines`, перевыполнить
  `3_lfo.scd`.
- **MIDI**: добавить строку в `~cfg_ctrl.map`, перевыполнить `5_control.scd`.
- **Таймлайн**: править `~cfg_seq.timeline`, перевыполнить `4_sequencer.scd`,
  затем `~seqStart.value`.
- **Эффекты**: править `~cfg_effects`, перевыполнить `6_effects.scd`.

Где что менять — по таблице §2 и по конвенции CONFIG (§6).

| `zeta`  | `synths/zeta.scd`  | §C.6 высокочастотное мягкое шумовое грейн-облако, HPF > 5 kHz (полная панель: окно фильтра + расстроенный резонансный банк `r1..r5` / текстура) | **реальный** (панель) |

| `theta`   | `synths/theta.scd`   | §C.8 9-голосный ультразвуковой шиммер 12–19 kHz, Splay | заглушка |

Оставшиеся заглушки (`beta`, `gamma`, `delta`, `epsilon`, `theta`) помечены
`// TODO: iterate this layer` и минимальны, но СЛЫШИМЫ и верны премисе синтеза
своего класса. `alpha` портирован полностью из `Tests/test_monade.scd`
(LOCKED — только чтение) и управляется панелью `~spec_alpha`. `zeta`
доведён из заглушки до полноценного слоя с панелью `~spec_zeta` (группы
`\freq`/`\tone`, рандом + пресеты — см. Приложение B). Шаблон панели общий
(0_lib), так что доводка следующего слоя = написать его `~spec_<class>`.

## 10. DSP-гигиена

- `Lag`/`.lag` на всех управляемых параметрах, способных прыгать (amp, freq,
  send, rq, частоты LFO и т.д.).
- `LeakDC` после суммирующих/нелинейных стадий (в каждом слое и в мастере).
- Частоты осцилляторов клампятся ниже Найквиста (`SampleRate.ir * 0.45`)
  там, где частота управляема пользователем (beta, gamma).
- Мастер: `Limiter(0.95)` (§K.4) после `LeakDC`.
- `theta` строго 12–19 kHz (клампится), `LPF(20 kHz)`.
- `zeta`: края фильтра `hpf`/`lpf` и частоты резонансного банка `r1..r5`
  клампятся в безопасные ВЧ-диапазоны; вклад резонанса ограничен `resAmt`
  (Ringz не «разгоняется»), панорама — `±0.4` (§F).
- Идемпотентная очистка: `Tdef/Pdef/MIDIdef/Ndef.all`, освобождение шин,
  `s.freeAll`. После Cmd-. и после `~appCleanup` — нет осиротевших узлов.
```

## Приложение A. Разбор слоя `alpha` (сигнал-флоу)

`Ndef(\alpha)` (`App/synths/alpha.scd`) — физмодель «связка из 10 пружин
в масле», портированная из `Tests/test_monade.scd`. Та же DSP-логика, но как
JITLib-слой (без gate-огибающей, без собственного посыла в эффекты).

```
ИСТОЧНИКИ ВОЗБУЖДЕНИЯ
                    sway = LFNoise2(0.1)  ── общий медленный «вдох/выдох»
                              │ модулирует density и общий уровень
                              ▼
  Dust2(density·sway) ──► contacts ── случайные импульсы соприкосновений
        │                     │
        │   ┌─── ticks: Decay2 → ×BrownNoise → HPF 2.8к   ┐ «сухой контакт»
        │   │    + metal: Mix(Ringz[3.1/4.7/6.3к]) ×0.16  │ (яркий, без низа)
        │   └──── × contact ──────────────────────────────┘
        │
  BrownNoise(0.07) ──► body ── непрерывный низкий гул (масса связки)

РЕЗОНАТОР — Array.fill(10): по одной «пружине» на тон
  для каждой пружины i:
     нота n_i ──.midicps──► sf (частота, Гц)
     voice = (body + contacts·collGain) · случ.амплитуда · 1/sd
        │
        ├─ ×3 AllpassC  ← дисперсия (disperse): металлический «зуммер»
        │
        └─ CombC(delay = 1/sf, decay = sd) ← РЕЗОНАНС на высоте тона
                │  (1/sf даёт высоту; sd — время звона)
                └─ × ringGain (низкие пружины громче → фундамент)

СУММА И ОБРАБОТКА
  bundle = Mix(10 пружин)/10 ·3.4  +  ticks·1.5
        ▼
  LPF(cutBase = 5400 − oil·3800)   ← масло (oil) затемняет
        ▼
  tanh (мягкое насыщение) → LeakDC  ← убрать DC после нелинейности

СТЕРЕО (приглушение дальнего конца связки)
  sigR = bundle                                   ← правый: сухой
  sigL = LPF(·(1−muffle)) → DelayC → ×(1−muffle)  ← левый: глуше/дальше
        ▼
  оба × amp × (0.78 + sway·0.22)   ← общий уровень + дыхание
        ▼
  [sigL, sigR]  ──► выход Ndef (его читает 2_mixer.scd)
```

### Три источника
1. **`Dust2`** — случайные импульсы соприкосновений пружин (плотность =
   `density`, модулирована `sway`).
2. **`ticks` / `metal`** — из тех же импульсов: яркий «сухой контакт».
   `HPF 2.8 кГц` убирает низ, три `Ringz` (3.1/4.7/6.3 кГц) добавляют
   металлический призвук. Громкость = `contact`.
3. **`body`** — непрерывный `BrownNoise`: гул массы связки в движении.

### Сердце тембра — 10 пружин
Цикл `Array.fill(10)`: каждая пружина возбуждается источниками и проходит
- каскад `AllpassC` ×3 — **дисперсия** (`disperse`), металлический «зуммер»;
- `CombC` — задаёт **высоту тона** (задержка `1/sf`) и **звон** (`decay`).
Низкие пружины громче (`ringGain`) — держат фундамент.

### Среда и пространство
- **`oil`** (масло) — опускает срез `LPF` (затемнение) и укорачивает звон Comb.
- **`muffle`** — приглушает левый канал (фильтр + задержка) → эффект глубины
  / удаления дальнего конца связки.
- Финал: `tanh` (мягкое насыщение) → `LeakDC` → стерео × `amp` × дыхание.

### Отличия от теста `Tests/test_monade.scd`
- Нет gate-огибающей `Env.asr` — включение/выключение делает микшер через
  `Ndef(\alpha).fadeTime = 6` (идиома JITLib «слой вкл/выкл», не play/release).
- Нет собственного `Out.ar(fx_bus,…)` — слой только возвращает `[sigL, sigR]`;
  посыл в реверб делает `2_mixer.scd` (§4).
- `amp` снижен 0.4 → 0.22 (gain staging под общий реверб, §K.4).
- Параметры идут через `Ndef(\alpha).set(...)` из CONFIG-блока (§6), их же
  дёргают `3_lfo.scd` (LFO) и `5_control.scd` (MIDI).

## Приложение B. Разбор слоя `zeta` (панель управления)

`Ndef(\zeta)` (`App/synths/zeta.scd`) — высокочастотное мягкое облако
шумовых зёрен (§C.6). Доведён из заглушки до полноценного слоя по образцу
`alpha`: единый источник правды `~spec_zeta` (19 параметров) → из него
0_lib строит слайдеры (`~openTuner.(\zeta)`), два рандомайзера и round-trip-
пресеты (`presets/zeta/` — стартовые `pure_rustle`, `glass_shimmer`).

```
ИСТОЧНИК ЗЁРЕН
  Dust(density · LFNoise2(densModRate)) ── мягко плавающая плотность
        │                                  (densModDepth — глубина дрейфа)
        ▼
  Decay2(grainAtk, grainDur) × WhiteNoise ── зерно: перкуссионная вспышка
        │                                  (grainAtk: мягкость ↔ щелчок)
        │
ОКРАСКА
        ├── Ringz[r1..r5 · spread] ── расстроенный резонансный банк (шиммер),
        │        подмешан по resAmt   (resAmt=0 → чистый шум, верно премисе)
        ├── bite: Decay2(коротк.) → BPF ── жёсткий ВЧ-транзиент («царапина»)
        ▼
ОКНО ФИЛЬТРА
  HPF(hpf) → LPF(lpf) ── хрупкая высокочастотная яркость
        ▼
СТЕРЕО
  Pan2(LFNoise1(panRate) · ±panWidth)  ── §F: pan ≤ ±0.4, медленно дышит
        ▼
  × amp → LeakDC ──► выход Ndef (его читает 2_mixer.scd)
```

### Две группы параметров (spec)
- group `\freq` — **частотные составляющие** (рандом «частоты»): у шума нет
  «нот», поэтому их роль играют края окна фильтра `hpf`/`lpf` и расстроенный
  резонансный банк `r1..r5` (Ringz), подкрашивающий облако.
- group `\tone` — **текстура/среда** (рандом «параметры»): `amp`, `density`,
  `grainDur`, `grainAtk`, `densModRate`/`densModDepth`, `panWidth`/`panRate`,
  `resAmt` (сухой шум ↔ тюнингованный шиммер), `resRing`, `spread`
  (расстройка банка → ширина), `bite` (жёсткость транзиента).

### Верность премисе §C.6
По умолчанию `resAmt`/`bite` низкие — слой звучит как мягкий ВЧ-шорох, не
жёсткое царапанье; подняв их в панели, получаем тюнингованный звенящий
шиммер и жёсткую «царапину». Reverb send держать НИЗКИМ (хрупкая яркость),
`fadeTime = 0.5` (быстрая текстура, не дрон).
