# Контракт файла слоя — `App/synths/<class>.scd`

> Единственный канонический источник требований к файлу звукового слоя.
> Агенты (`supercollider-dev`, `supercollider-analyst`) ссылаются СЮДА и не
> носят собственных копий — при изменении конвенции правится один файл.
> Эталонные реализации: `App/synths/alpha.scd`, `App/synths/zeta.scd`.

## 1. Структура файла

1. **РОВНО ОДИН верхнеуровневый блок `( … )`** — CONFIG и LOGIC живут внутри
   него. `1_app.scd` грузит модули через `.load`, а `.load` выполняет только
   ПЕРВЫЙ блок файла: второй блок молча не выполнится, а чаще файл вообще не
   скомпилируется («unexpected '(', expecting end of file»).
2. **CONFIG сверху** (`// ===== CONFIG — edit here =====`), LOGIC ниже
   (`// ===== LOGIC =====`). Поведение меняется правкой верха файла.
3. **Каждый параметр CONFIG обязан реально влиять на Ndef.** «Мёртвый CONFIG»
   (значение объявлено сверху, но захардкожено литералом в теле) запрещён.

## 2. Спецификация параметров (`~spec_<class>`)

Слой с панелью управления задаёт единый источник правды:

```supercollider
~spec_<class> = (
    name: \<class>,
    params: [
        ( key: \amp, group: \tone, spec: ControlSpec(lo, hi, warp, step, default),
          unit: " Hz", disp: \note ),   // unit/disp — опциональны
        ...
    ]
);
```

- `group: \freq` — частотные составляющие (рандомайзер «частоты»);
  `group: \tone` — текстура/тембр/среда (рандомайзер «параметры»).
- `disp: \note` — значение УЖЕ в MIDI-нотах; `disp: \hz` — значение в Гц,
  рядом показывается ближайшая нота+центы. `disp` ставится только на
  `\freq`-строки.
- **Каждая строка spec ↔ аргумент `Ndef` с тем же дефолтом** (и наоборот).
- Заглушка без панели использует упрощённый `~cfg_<class>` + `Ndef.set` —
  но правило «нет мёртвого CONFIG» действует и для неё.

## 3. Хвост регистрации (дословно, после тела Ndef)

```supercollider
~ungSpecs = ~ungSpecs ? ();
~ungVals  = ~ungVals  ? ();
~ungSpecs[\<class>] = ~spec_<class>;
~ungVals[\<class>]  = ();
~spec_<class>.params.do { |p|
    Ndef(\<class>).set(p.key, p.spec.default);
    ~ungVals[\<class>][p.key] = p.spec.default;
};
Ndef(\<class>).fadeTime = <…>;
```

`fadeTime` по роли слоя: Grund/континуум — 3–8 s; событийные и быстрые
текстуры — 0.5–1 s.

## 4. Жизненный цикл и маршрутизация

- **БЕЗ** `s.waitForBoot`, `s.sync`, cleanup-блока — жизненный цикл ведёт
  `1_app.scd`; слой не освобождает свой прокси.
- **БЕЗ** `Out.ar`/`ReplaceOut` — слой возвращает СТЕРЕО в свой Ndef-выход;
  маршрутизацию и посылы в эффекты делает микшер (`2_mixer.scd`).
- Без per-event Synth/`{}.play` — все импульсы живут внутри персистентного
  графа (`Dust`/`Decay2`/`Ringz` и т.п.).

## 5. DSP-гигиена

- Nyquist-кламп ПОСЛЕ lag на каждой управляемой частоте:
  `freq.lag(0.2).clip(lo, SampleRate.ir * 0.45)` (не хардкод 20000).
- `.lag` на всех прыгающих контролах (amp 0.05; фильтры 0.2; плотности 0.3).
- `LeakDC.ar` в конце (после суммы/нелинейности/импульсного контента).
- Панорама §F: `panWidth ≤ ±0.4`, `panRate` медленный (`LFNoise1.kr ≤ 2 Hz`).
- Имена — греческая буква ASCII: файл = `Ndef(\<class>)` = `~spec_<class>`;
  немецкие имена (`monade`, `riß`, …) — только как ссылка на §C.

## 6. Project footguns (SC/Qt — пойманы в этом репозитории)

- **Ключи Event, совпадающие с методами** — доступ только скобками:
  `p[\group]`, `block[\set]`, `e[\key]`. Точечный доступ молча вызывает метод
  Object/Event, а не читает ключ.
- **`win = dict[k] = Window(...)`** — присваивание в словарь возвращает
  СЛОВАРЬ, и `win` биндится не на окно. Присваивать раздельно.
- **Отложенный onClose старого окна** при close→reopen затирает реестры
  нового — guard `if(~slot === win) { … }` (см. 0_lib и 2_mixer).
- **`.close` уже разрушенного Qt-объекта** бросает ошибку и обрывает сборку —
  всегда `try { win.close }`.
- **`Ndef.all`** — словарь «имя сервера → ProxySpace», его ключи НЕ ключи
  прокси. Идемпотентную очистку вести по собственному реестру (см. 3_lfo).
- **`Array:includes` для String** сравнивает по идентичности — членство строк
  проверять `.any { |x| x == s }`.
- **Тяжёлое Qt-окно**: `front` сразу после сборки может не показать окно —
  один `.defer(0.3)` после раскладки (см. 0_lib).
